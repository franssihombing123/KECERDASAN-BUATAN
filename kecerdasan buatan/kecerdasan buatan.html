Sistem Diagnosa Jantung Berbasis Kecerdasan Buatan

Model Intelegent

Agent Intelegent ini merupakan sebuah sistem yang dapat melakukan diagnosa penyakut jantung berbasis kecerdasan buatan, dimana sistem ini akan melakukan identifikasi, dan pemecahan masalah yang nantinya sistem ini akan menampilkan diagnosa yang diderita oleh pasien.

adapun tujuan atau fungsi dari agent intelegent ini untuk melakukan rancangan dan mengimplementasikan sistem diagnosis penyakit jantung online berbasis website. Sistem diagnosis menggunakan artificial intellegent dan memasukkan data dari pengguna berdasarkan beberapa pertanyaan yan diajukan oleh sistem, dimana cara kerja dari sistem i ini memanfaatkan media elektronik dan perlu adanya gawai ataupun perangkat elektronik lainnya seperti komputer dan sebagainya yang menjadi media bagi pengguna dalam menjalankan sistem ini

Persepsi

Persepsi dapat diartikan sebagi berikut "Suatu proses merubah sesuatu dan lingkungan ke dalam indra penerima(Memori, otak dan lain-lain)" Dalam kata lain perception disini adalah apa saja yang menjadi input pada agent yang menjadi media bagi pengguna dalam menjalankan sistem ini.
Intelligent agent dalam diagnosis penyakit jantung yang dimana menggunakan aplikasi website sebagai media mendiagnosisnya, memiliki inputan (Perception) berupa keyboard dan mouse berfungsi sebagai memilih gejala penyakit jantung yang dialami oleh user sehingga website dapat mendiagnosis penyakit yang dialami

Sekuen Persepsi

Ketika merancang sebuah agent, kita gharus mendefenisikan masalah (task environment) yaitu PAGE (Russel & Norvig menggunakan singkatan PEAS).
Percept: Apa saja yang jadi input si agent?
Action: Apa saja yang bisa dilakukan si agent?
Goals: Apa tujuan si agent?
environment: Dimanakah si agent berperan


Agent function
Sebuah fungsi yang memetakan sejarah input (percept sequence) terhdapa tindakan yang dilakukan (action) (f.P*A). Metode CBR yang digunakan pada agent yang kami ambil menggunakan data kategori penyakit jantung beserta defenisi, gejala, dan tindakan medis y ang diambil untuk menarik hubungan antara data tersebut. Masukkan gejala dari pengguna kemudian disesuaikan oleh aplikasi terhadap aturan hubungan yang ada dan kemudian ditampilkan dalam bentuk diagnostik online


Agent program
Sebuah program yang mengimplementasikan fungsi f di atas arsitektur
=>Agent = arsitektur + program
=> Agent program menerima input percep terakhir
Untuk setiap urutan persepsi(percept sequence) yang ada, agen rasional harus memilih tindakan yang diharapkan untuk memaksimalkan ukuran kinerjanya.
Agent yang kami gunakan adalah Sistem Diagnosis Penyakit Jantung
Percept: Keyboard dan mouse(msauknya inputan gejala, temuan, dan jawaban pasien)
=> Percept Sequence (urutan inputan):
{[Pengguna, Dada terasa penuh]},
{[Pengguna, Detak jantung cepat ]},
{[Pengguna, Detak jantung lambat]},
{[Pengguna, Nyeri pada dada]},
{[Pengguna, Panas]},
{[Pengguna, Sesak Napas]},
{[Pengguna, Katup jantung tidak bekerja dengan baik]},
{[Pengguna, Bunyi jantung abnormal]},
{[Pengguna, Kebocoran atau penyempitan]},
{[Pengguna, Badan lesu atau lemah]},
{[Pengguna, Pusing]},
{[Pengguna, Pingsan]},
{[Pengguna, Katub jantung tanpa lubang]},
{[Pengguna, Sulit tidur]},
{[Pengguna, Denyut nadi yang lemah dan cepat]},
............. dan seterusnya
Action: Layar Display: Menanyakan dengan memberikan perrtanyaan baik dalam bentuk formulis ataupun checklist formulir, memberikan jawaban diagnosis kepada pengguna terkait input yang telah diberikan berbentuk teks, memberikan pengobatan, rujukan dan solusi.
Goal: Tujuan mengetahui penderita penyakit jantung, mengetahui potensi terkena penyakit jantung atau tidak, mengetahui tingkat penyakit penderita.
Environment:Didalam semua perangkat yang terhubung dengan koneksi internet karena agent berbasis website.
